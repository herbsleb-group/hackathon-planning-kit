The community building hackathons are used to develop resources, infrastructure, practices, and culture for community. However, it is not possible to achieve mutliple orthogonal goals with one-off events. For example, hackathon for building a great technical artifact might not be possible if people want to learn at the same time. It is therefore important for organizers to consider holding recurring events at specified intervals or as the need is perceived to facilitate the sustained evolution of community hacking ecosystem.

The following tree with activated links is a great example of hackathon for community building. In this tree, - the root node represents the goal, - all interior nodes are decision nodes and contain high-level features (e.g., projects, teams, process, mentor, and competition) - the leaf nodes contain low-level features, and - the activated links colored in blue show the paths to the important features that the organizers need to focus on but the inactivated links should not be discarded.
